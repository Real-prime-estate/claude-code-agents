저수준에서 헬퍼(공용 래퍼 함수, 작은 컴포넌트 추상화, View 확장 등)를 미리 만들어두는 것을 사용자가 싫어한다. 코드를 직접 풀어서 쓰고, **명백히 반복이 증명된 뒤에만** 헬퍼로 추출한다. 고수준(아키텍처 레벨) 추상화에 대한 입장은 아직 미정 — 저수준에 한정된 지침.

**Why:** 사용자가 "저수준에서 헬퍼 만들어서 좋은 꼴 못 봤다"고 명시. 선제적 헬퍼는 잘못된 추상화로 굳어 디버깅·수정을 더 어렵게 만든 경험이 있음. (특히 SwiftUI처럼 런타임 동작이 암묵적인 곳에서 헬퍼는 문제 위치를 흐림.)

**How to apply:** 화면/기능을 구현할 때 인라인으로 직접 작성한다. "공용으로 빼면 깔끔하다"는 이유만으로 헬퍼/컴포넌트/확장을 만들지 말 것. 세 번 이상 동일 패턴이 실제로 반복되고 사용자가 동의할 때만 추출 제안. 제안 시에는 추출 근거(반복 횟수)를 함께 댄다. JH iOS 재구축은 "한 번에 한 화면, 매 단계 시뮬레이터 검증"으로 진행 — [[jh-ios-rebuild-incremental]] 참고.
