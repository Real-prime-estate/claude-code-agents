데이터 모델 변경·1급 객체 도입·*완성도 우선* 명목 결정을 할 때 다음 체크리스트로 자기심문할 것. 안 하면 *시스템 설계자 본능*에 끌려 같은 실수 반복.

**Why:** 2026-05-03 *세 차례* 정정 라운드 — (1) 사용자 도전으로 사건·진실 객체화 폐기 → (2) "MVP bias" 지적에 *반대 방향 bias*로 객체화 복원 → (3) 사용자 재도전 "왜 객체화?"로 다시 폐기. 두 번째 정정 자체가 *반대 방향 bias*였음. think-deep 토론 라운드 2에서 비판 에이전트가 *Q5 작가 의도 체크리스트*를 *진실 객체 우회 도입*으로 평가하여 wounded — 사용자가 "후순위" 결정으로 정합성 보존.

**How to apply:** 데이터 모델·객체화·"완성도" 단어가 결정에 등장하는 모든 라운드.

## 체크리스트

### ① "완성도 = 객체화" 등식을 의심
- *완성도*는 작가가 *작품을 완성하는가*. 도구가 *모든 의도를 객체로 알아야* 완성도가 아님
- 작가는 외부 메모·산문으로도 작품을 완성 가능
- 데이터 모델 차원의 명세화 ≠ 도구의 완성도

### ② 시스템 설계자 본능 (CS 편향) 점검
- "구조화된 데이터가 좋다"는 본능은 *작가 도구*에 직선 적용 시 위험
- 작가 도구 표준 패턴: *부드러운 구조화* (Notion·World Anvil·Inform 7·Twine — 모두 구조 + 자유 텍스트 균형)
- "도구가 모든 것을 읽을 수 있어야" 본능 의심

### ③ Bias 정정 시 *반대 방향 bias* 주의
- "이 결정은 X bias"라는 지적을 *X의 반대 = 정답*으로 단정 금지
- *공정한 평가*가 정답이지 *반대 방향*이 정답이 아님
- 자존심으로 *이전 결정을 잘못으로 단정*하고 *반대 결정 복원*하는 패턴 경계

### ④ 효용 정량화 없는 객체화는 폐기

객체화·1급 필드·새 데이터 모델 도입 주장 시 다음을 모두 답할 수 있어야:
- *지금* 어떤 게임 인터랙션·검증·시뮬레이션이 *이 객체를 요구*하는가?
- *미래 예상 효용*이 아닌 *현재 명시 효용*은?
- 그 효용은 *외부 대안*(텍스트 자유 입력·작가 외부 메모·기존 모델 활용)으로 *불가능*한가?

답이 "후순위·미정·외부 대안 가능"이면 객체화 폐기.

### ⑤ "사용자가 동의했다" 함정
AUQ에서 사용자가 *추천 옵션*을 선택했다고 *그 결정이 정당*한 건 아님. 사용자도 내 추천에 끌릴 수 있고 *재고 시점*이 후속에 옴. 추천 자체의 정당성을 내가 먼저 점검.

### ⑥ 우회 도입 의심
"옵셔널 필드라 영향 0이니 도입해도 무방" 식 논리는 *우회 도입* 함정. 진실 객체 폐기 결정 후 *작가 의도 체크리스트* 같은 *형식만 다르고 의미 같은* 객체를 도입하면 결정 *정신* 위반. 의미 동등 검토.

## 정정의 정정

여러 라운드 정정에서 *어느 라운드의 결정이 옳은가*는:
- *효용 정량화*가 명시된 결정이 옳음
- *명목적 단어*(완성도·본질·정당성)에 끌린 결정은 의심
- *외부 대안 점검*이 통과한 결정이 옳음
- *시스템 설계자 본능*과 *작가 현장 가치*를 분리한 결정이 옳음

## 적용 예 (2026-05-03 본 사건)

- 사건(crime) 객체화: 게임 인터랙션 없음 + 텍스트 두 필드 대안 가능 → ❌ 폐기
- 진실(truths) 객체화: 게임 동작 효용 0 + 외부 메모 가능 + 자동 검증은 후순위 → ❌ 폐기
- 단서→진실 매핑: 진실 객체 폐기에 동반 → ❌ 폐기
- Q5 작가 의도 체크리스트: *진실 객체 우회 도입* 의심 → ⏸️ 후순위 (외부 작가 M2 검증 후 결정)
- meta.synopsis·briefing: 게임 UI 슬롯(대기실·시작 화면)에 명시 박힘 + 텍스트 자유 입력으로 작가 효용 ↑ → ✅ 유지

미래에 객체화가 *진짜* 필요한 시점이 올 수 있음 — *그때*는 *효용 정량화*를 명시한 라운드가 되어야 정당.
