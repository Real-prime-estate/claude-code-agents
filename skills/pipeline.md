# Pipeline: think-deep → code 자동화

think-deep 보고서의 합의 권고를 파싱하여, 조건에 따라 자동 또는 수동으로 실행 에이전트에 전달합니다.

## 인자 파싱

사용자 입력: `$ARGUMENTS`

```
/pipeline <report.md 경로>
/pipeline --dry-run <경로>     (분류만 하고 실행 안 함)
```

## 실행 프로토콜

### Phase 0: 사전 검증

1. report.md 존재 확인 (Read)
2. git 상태 확인:
   ```
   Bash: git status --porcelain
   ```
   dirty working tree이면 경고: "커밋되지 않은 변경이 있습니다. 계속하시겠습니까?"

### Phase 1: 보고서 파싱

1. Read: report.md
2. "합의된 권고" 섹션에서 각 항목 추출:
   - 항목명
   - 합의도 (3/3, 2/3, 1/3)
   - 평균 확신도 (%)
   - 예상 LOC (있으면)
   - 검증 방법 (있으면)
   - 리스크 (있으면)

### Phase 2: 분류

각 항목을 분류한다:

```
자동 실행 조건 (AND):
  - 합의도 = 3/3 (만장일치)
  - 평균 확신도 ≥ 85%
  - LOC ≤ 50 (소규모 변경)

사용자 확인:
  - 위 조건 미충족

스킵:
  - report.md에서 "실험적 검증 필요"로 표기된 항목
  - 철회된 제안
```

### Phase 3: 사용자 확인

분류 결과를 표시한다:

```markdown
## 자동 실행 예정 (N건)
1. omega 수정 (3/3, 90%, 3 LOC) → C executor
2. dead code 제거 (3/3, 92%, -45 LOC) → C executor

## 사용자 확인 필요 (M건)
3. [ ] 멀티그리드 도입 (2/3, 60%, ~400 LOC)
4. [ ] MCI 조건부 통합 (2/3, 63%, ~55 LOC)

## 스킵 (K건)
5. 4K A/B 실험 (실험적 검증 필요)

진행하시겠습니까?
```

사용자가 확인 항목 중 실행할 것을 선택하면 진행.
`--dry-run`이면 여기서 종료.

### Phase 4: 프로젝트 분석가 연동

1. `.claude/analysis.md` 존재 확인
2. 있으면 Read하여 컨텍스트로 준비
3. 없으면 `/analyze --brief` 자동 실행

### Phase 5: Executor 실행

각 항목에 대해:

1. 언어 감지 (report.md에 언어가 명시되어 있으면 사용, 없으면 자동 감지)
2. 해당 executor 에이전트 스폰:

```
Agent(
  subagent_type="agents:{언어}-executor",
  description="{언어} executor: {항목명}",
  prompt="
    ## 작업 지시
    {report.md에서 추출한 권고 내용}

    ## 프로젝트 컨텍스트
    {analysis.md 요약}

    ## 근거
    합의도: {N/3}
    확신도: {N%}
    근거: {report.md에서 추출한 근거}
    리스크: {report.md에서 추출한 리스크}

    ## 검증 방법
    {report.md에서 추출한 검증 방법}

    ## 주의사항
    - 상세 컨벤션 파일을 반드시 Read
    - 자체 리뷰 체크리스트 수행
    - 한국어로 보고
  "
)
```

3. 순차 실행 (파일 충돌 방지). 다만 서로 다른 파일을 수정하는 경우 병렬 가능.

### Phase 6: 실행 보고서 생성

Write 도구로 report.md와 같은 디렉토리에 `pipeline-report.md` 저장:

```markdown
# Pipeline 실행 보고

> 원본: {report.md 경로}
> 실행일: {날짜}

## 자동 실행 완료 (N건)
| # | 권고 | 언어 | 결과 | 컨벤션 리뷰 | 수정 파일 |
|---|------|------|------|------------|----------|
| 1 | {항목} | C | 완료 | 위반 0건 | {파일} |

## 사용자 선택 실행 (M건)
| # | 권고 | 언어 | 결과 | 컨벤션 리뷰 | 수정 파일 |
|---|------|------|------|------------|----------|

## 스킵 (K건)
| # | 권고 | 이유 |
|---|------|------|

## 다음 단계
{실험 필요 항목, 미실행 항목에 대한 권고}
```

## 복잡도 자동 판단

**실행**: "토론 결과 구현해", "권고 실행해", "파이프라인 돌려"
**무시**: 직접 코드 작성 요청 (→ /code), 토론 요청 (→ /think-deep)
